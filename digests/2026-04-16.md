## AI Agents & Frameworks

**GenericAgent — Self-evolving agent with a skill tree**
An AI agent that grows a reusable skill tree from a small seed codebase, slashing token usage while gaining full system control. Interesting because it tackles the core inefficiency of current agents: they never learn from past tasks. [GitHub](https://github.com/lsdefine/GenericAgent)

**Evolver — GEP-powered self-evolution engine for AI agents**
Uses Gene Expression Programming so AI agents can autonomously improve their own strategies after deployment. A compelling alternative to manual retraining that borrows from evolutionary computation. [GitHub](https://github.com/EvoMap/evolver)

**openai-agents-python — Lightweight multi-agent workflow framework**
OpenAI's own minimal Python framework for orchestrating multi-agent systems with tool use, handoffs, and guardrails. Worth studying as the canonical pattern for agent coordination from the source. [GitHub](https://github.com/openai/openai-agents-python)

**open-agents — Vercel template for cloud-hosted AI agents**
A deploy-ready Next.js template for building and hosting AI agents with minimal setup. Removes the infrastructure yak-shaving so you can focus on agent logic. [GitHub](https://github.com/vercel-labs/open-agents)

**Cognee — Knowledge graph engine for AI agent memory**
Gives agents persistent, structured memory via knowledge graphs in a few lines of code. Solves the amnesia problem that makes agents repeat mistakes across interactions. [GitHub](https://github.com/topoteretes/cognee)

## Claude Code Ecosystem

**andrej-karpathy-skills — CLAUDE.md for better Claude Code behavior**
A curated prompt file encoding Karpathy's observations to suppress common LLM coding mistakes. A weekend project that could meaningfully improve your daily Claude Code output. [GitHub](https://github.com/forrestchang/andrej-karpathy-skills)

**claude-mem — Automatic memory plugin for Claude Code**
Captures, compresses, and re-injects session context so Claude remembers past work between sessions. Directly addresses the most frustrating limitation of agentic coding assistants. [GitHub](https://github.com/thedotmack/claude-mem)

**android-reverse-engineering-skill — Claude Code skill for Android RE**
Equips Claude Code with specialized knowledge for Android reverse engineering using jadx, apktool, and frida. A nice example of creating vertical domain skills for general-purpose AI tools. [GitHub](https://github.com/SimoneAvogadro/android-reverse-engineering-skill)

## Dev Tools

**Fakecloud — Free, open-source AWS emulator (Go)**
A local AWS emulator for building and testing cloud apps without incurring costs. Positioned as the truly-free alternative to LocalStack's increasingly gated free tier. [GitHub](https://github.com/faiscadev/fakecloud)

**Hiraeth — AWS emulator (Rust)**
Another local AWS emulator, this one built in Rust for speed. Two competing open-source AWS emulators trending the same day tells you there's real demand here. [GitHub](https://github.com/SethPyle376/hiraeth)

**wacli — WhatsApp CLI**
Send and read WhatsApp messages from your terminal or scripts. Simple, useful, and surprisingly nonexistent until now — unlocks WhatsApp for automation and developer workflows. [GitHub](https://github.com/steipete/wacli)

## AI/ML Infrastructure & Research

**Libretto — Making AI browser automations deterministic**
Records AI-driven browser interactions and replays them deterministically, turning flaky LLM automations into reliable scripts. Solves the "works 80% of the time" problem blocking AI agents in production. [GitHub](https://github.com/saffron-health/libretto)

**DFlash — Block diffusion for speculative decoding**
Applies block diffusion to speculative decoding to dramatically accelerate LLM inference. Targets the core latency bottleneck of autoregressive generation. [GitHub](https://github.com/z-lab/dflash)

**Magika — AI-powered file content type detection (Google)**
A lightweight deep learning model that outperforms magic-byte file detection. Useful anywhere file uploads or processing happen — security, pipelines, storage systems. [GitHub](https://github.com/google/magika)

**Voicebox — Open-source voice synthesis studio**
A desktop app for creating and managing AI-generated voices with a studio interface. Fills the gap left by proprietary TTS tools that are expensive and locked down. [GitHub](https://github.com/jamiepine/voicebox)

**Dive into LLMs — Hands-on LLM programming tutorials**
Code-first tutorials teaching you to build and understand LLMs from scratch. Not another theory dump — this is the "build it to understand it" resource the ecosystem needs. [GitHub](https://github.com/Lordog/dive-into-llms)

**Omi — Always-on screen and conversation monitoring assistant**
An AI assistant that watches your screen and listens to conversations to proactively surface action items and context. Ambitious and privacy-fraught, but the "ambient assistant" category is clearly heating up. [GitHub](https://github.com/BasedHardware/omi)

## Wild Cards 🃏

**Let AI Gamble — Claude with a real casino bankroll**
Gives Claude an API-connected casino account and lets it autonomously gamble until it goes broke or hits the context window. A weirdly compelling stress test of LLM reasoning under uncertainty — and a weekend build. [letaigamble.com](https://letaigamble.com/)

**MacMind — Transformer neural network in HyperCard on a 1989 Macintosh**
A working transformer implemented entirely in HyperTalk on vintage Mac hardware. Part art project, part education, fully absurd — and it proves modern ML concepts can be grasped on a machine with less RAM than a JPEG. [GitHub](https://github.com/SeanFDZ/macmind)
